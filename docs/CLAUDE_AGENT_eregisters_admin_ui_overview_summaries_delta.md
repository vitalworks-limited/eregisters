## 4. Non-Negotiable Performance Rules

The new Admin Overview must never run heavy DHIS2 tracker queries directly from the browser.

### 4.1 Prohibited frontend behavior

Do not allow any Admin Overview component, chart, map, or card to call these patterns directly:

```text
GET /api/42/tracker/trackedEntities?fields=*
GET /api/42/tracker/trackedEntities?...enrollments[*,events[*]]
GET /api/42/tracker/trackedEntities?...pageSize=100
GET /api/42/tracker/trackedEntities without updatedAfter for national summaries
GET /api/42/tracker/events for national summary counts without cache/materialization
POST /api/42/tracker?async=false for bulk monitoring or administrative operations
Any all-facility browser-side loop over tracker endpoints
Any browser-side log parsing from Tomcat/PostgreSQL/Catalina files
```

### 4.2 Allowed data sources

The Admin Overview may read from these safe sources:

1. eRegisters cached summary endpoint if available.
2. DHIS2 dataStore namespace containing precomputed summary JSON.
3. Static JSON summary generated by a server-side cron/summarizer and deployed behind the app.
4. Lightweight DHIS2 metadata endpoints with explicit fields and pagination.
5. DHIS2 analytics endpoints only when they are already computed and safe.
6. Local app telemetry summaries, aggregated before upload.
7. Server-side summarizer output for Tomcat access logs, DHIS2 jobconfiguration, Hikari warnings, PostgreSQL connection pressure, and Catalina program-rule errors.

### 4.3 Mandatory safe-query guard

Add a guard layer around the app API client for Admin Overview calls.

The guard must detect and block unsafe URL patterns before the request leaves the browser.

Create something equivalent to:

```ts
export function assertAdminOverviewSafeRequest(url: string, context: string): void {
  const decoded = decodeURIComponent(url);

  const isTrackerTrackedEntities = /\/tracker\/trackedEntities/.test(decoded);
  const isTrackerEvents = /\/tracker\/events/.test(decoded);
  const hasFieldsStar = /fields=\*/.test(decoded);
  const hasNestedEvents = /enrollments\[[^\]]*events\[\*\]/.test(decoded) || /events\[\*\]/.test(decoded);
  const hasPageSize100 = /pageSize=100/.test(decoded);
  const isBulkTrackerPostSync = /\/tracker/.test(decoded) && /async=false/.test(decoded);

  if (context === 'ADMIN_OVERVIEW') {
    if (isTrackerTrackedEntities || isTrackerEvents || hasFieldsStar || hasNestedEvents || hasPageSize100 || isBulkTrackerPostSync) {
      throw new Error(`Unsafe Admin Overview request blocked: ${decoded}`);
    }
  }
}
```

The final implementation must adapt this to the actual codebase architecture.
